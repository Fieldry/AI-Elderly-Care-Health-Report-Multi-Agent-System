# AI 养老健康评估系统 - 多轮对话问题与数据需求文档

## 文档说明

本文档整理了系统在多轮对话中向用户提出的所有问题，以及需要收集的用户数据字段。用于后续问题设计和优化。

---

## 一、对话流程概述

系统采用**分组追问**的方式，将用户画像分为 **10 个部分**，每部分包含若干相关字段。系统会：

1. 首先显示欢迎语，**避免提及"10个部分"**，以免老年人觉得要填写很多内容而放弃
2. 逐个分组提问，**每次只问1-2个问题**，用户可以用自然语言回答
3. 系统自动提取结构化数据，**支持模糊表述**（如"以前有，现在没有"）
4. 所有信息收集完毕后，进入确认阶段
5. 用户确认后，生成健康评估报告

**总字段数**：62 个字段

**设计原则**：
- ✅ 温和交互，避免问卷感
- ✅ 大白话表述，避免专业词汇
- ✅ 拆分提问，每次1-2个问题
- ✅ 支持模糊回答和口语化表达

## 二、10 个问题分组详细说明

### 第 1 部分：基本信息（G1）

**字段数**：6 个

**开场话术**：
> 您好😊 我想先了解一下您的日常身体和生活情况，这样才能给您整理一份更贴合您实际情况的健康建议。我会一步一步来问，都是些日常问题，您按平时说话的方式回答就行。要是有些记不清，也没关系，大概说一下就可以。

**提问内容**（拆分为多轮，每次1-2个问题）：

**第1轮**：
> 先来了解一下基本情况吧😊 老人家今年多大了？是男是女？

**第2轮**：
> 住在哪个省或城市？是城市还是农村？

**第3轮**：
> 以前上过几年学？现在的婚姻状况是怎样的（在婚、丧偶还是其他）？

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `age` | 年龄 | 整数 | 如：82 |
| `sex` | 性别 | 文本 | 男 / 女 |
| `province` | 省份/地区 | 文本 | 如：北京、河南、广东 |
| `residence` | 居住地类型 | 文本 | 城市 / 农村 |
| `education_years` | 读过几年书 | 整数 | 如：6（小学）、0（没读过书） |
| `marital_status` | 婚姻状况 | 文本 | 在婚 / 丧偶 / 离婚 / 未婚 |

---

### 第 2 部分：健康限制（G2）

**字段数**：1 个

**提问内容**：
> 这半年里，身体有没有影响您出门、活动或者做事？比如走路不利索、做家务更吃力了，或者不太敢自己出门？

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `health_limitation` | 过去6个月身体是否影响了日常活动 | 文本 | 没有 / 有一点 / 比较严重 |

**说明**：这是一个重要的健康指标，用于初步判断老人的整体健康状况。

---

### 第 3 部分：日常活动能力 - 基本生活（BADL）（G3）

**字段数**：6 个

**提问内容**：
> 接下来问一下日常生活的基本动作。我列出来，您可以直接说哪些需要帮助，哪些能自己来👇
> 
> ① 洗澡  ② 穿衣  ③ 上厕所  ④ 室内走动（起床、坐椅子这类）  ⑤ 大小便控制  ⑥ 吃饭
> 
> **回答示例**：
> - "洗澡需要帮助，其他都能自己来"
> - "1、2、3需要帮助"
> - "1、2、5需要帮助，其他都行"

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `badl_bathing` | 洗澡能力 | 文本 | 能自己来 / 需要部分帮助 / 需要很多帮助 |
| `badl_dressing` | 穿衣能力 | 文本 | 能自己来 / 需要部分帮助 / 需要很多帮助 |
| `badl_toileting` | 上厕所能力 | 文本 | 能自己来 / 需要部分帮助 / 需要很多帮助 |
| `badl_transferring` | 室内活动（起床、坐椅）能力 | 文本 | 能自己来 / 需要部分帮助 / 需要很多帮助 |
| `badl_continence` | 大小便控制能力 | 文本 | 能自己来 / 需要部分帮助 / 需要很多帮助 |
| `badl_eating` | 吃饭能力 | 文本 | 能自己来 / 需要部分帮助 / 需要很多帮助 |

**说明**：BADL（Basic Activities of Daily Living）是评估失能状态的核心指标。支持用户用数字编号或自然语言回答。

---

### 第 4 部分：日常活动能力 - 工具性活动（IADL）（G4）

**字段数**：8 个

**提问内容**：
> 再问一些稍微复杂一点的日常活动。同样，您可以直接说哪些有困难，哪些能做👇
> 
> ① 串门/走亲戚  ② 买东西  ③ 做饭  ④ 洗衣服  ⑤ 走1公里路  ⑥ 提约5斤重的东西  ⑦ 蹲下再站起来  ⑧ 坐公共交通
> 
> **回答示例**：
> - "走远路和提重物有点困难，其他基本都能做"
> - "1、2、3能做，4、5、6有困难，7、8做不了"
> - "除了走远路，其他都还行"

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `iadl_visiting` | 串门/走亲访友能力 | 文本 | 能做 / 有点困难 / 做不了 |
| `iadl_shopping` | 购物能力 | 文本 | 能做 / 有点困难 / 做不了 |
| `iadl_cooking` | 做饭能力 | 文本 | 能做 / 有点困难 / 做不了 |
| `iadl_laundry` | 洗衣能力 | 文本 | 能做 / 有点困难 / 做不了 |
| `iadl_walking` | 步行1公里能力 | 文本 | 能做 / 有点困难 / 做不了 |
| `iadl_carrying` | 提重物（约5kg）能力 | 文本 | 能做 / 有点困难 / 做不了 |
| `iadl_crouching` | 蹲下站起能力 | 文本 | 能做 / 有点困难 / 做不了 |
| `iadl_transport` | 乘坐公共交通能力 | 文本 | 能做 / 有点困难 / 做不了 |

**说明**：IADL（Instrumental Activities of Daily Living）评估更复杂的日常活动能力。支持用户用数字编号或自然语言回答。

### 第 5 部分：慢性病情况（G5）

**字段数**：7 个

**提问内容**：
> 有没有被医生诊断过这些病？每项说有还是没有就好：
> 
> 高血压、糖尿病、心脏病、中风（脑卒中）、白内障、癌症、关节炎/风湿

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `hypertension` | 高血压 | 文本 | 是 / 否 |
| `diabetes` | 糖尿病 | 文本 | 是 / 否 |
| `heart_disease` | 心脏病 | 文本 | 是 / 否 |
| `stroke` | 中风/脑卒中 | 文本 | 是 / 否 |
| `cataract` | 白内障 | 文本 | 是 / 否 |
| `cancer` | 癌症/恶性肿瘤 | 文本 | 是 / 否 |
| `arthritis` | 关节炎/风湿 | 文本 | 是 / 否 |

**说明**：这些是老年人常见的慢性病，对健康风险评估至关重要。

### 第 6 部分：认知功能（G6）

**字段数**：6 个

**开场话术**：
> 接下来是几个很简单的小问题，主要想看看您最近记忆和反应怎么样。很多老人都会有点记不清，这都很正常😊

**提问内容**（拆分为多轮）：

**第1轮 - 时间定向**：
> 知道今天是几号吗？

**第2轮**：
> 知道现在是几月份吗？

**第3轮**：
> 知道现在是什么季节吗？

**第4轮**：
> 知道自己现在在哪里吗？

**第5轮 - 计算能力**（拆分成三道题）：
> 现在问您三道简单的减法题，不用太快，慢慢想：
> 
> ① 100减7等于几？
> 
> ② 那个答案再减7呢？
> 
> ③ 再减7呢？

**第6轮 - 画图能力**：
> 最后一个，能照着样子画一个简单的图形吗？比如重叠的五边形（我会给您看图）

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `cognition_time` | 时间定向（知道今天是几号） | 文本 | 正确 / 错误 / 不知道 |
| `cognition_month` | 月份定向（知道现在是几月份） | 文本 | 正确 / 错误 / 不知道 |
| `cognition_season` | 季节定向（知道现在是什么季节） | 文本 | 正确 / 错误 / 不知道 |
| `cognition_place` | 地点定向（知道自己在哪里） | 文本 | 正确 / 错误 / 不知道 |
| `cognition_calc` | 计算能力（100连续减7，做3次） | 列表 | ['正确','正确','错误'] 等 |
| `cognition_draw` | 画图能力（能否照样画一个简单图形） | 文本 | 正确 / 错误 |

**说明**：认知功能评估用于识别认知障碍和痴呆风险。每道题都拆分提问，避免一次性问太多导致老人漏答或选择性回答。

---

### 第 7 部分：心理状态（G7）

**字段数**：3 个

**提问内容**（拆分为多轮）：

**第1轮 - 抑郁情绪**：
> 最近这段时间，您会不会常常觉得心里闷闷的、高兴不起来？选从不、很少、有时、还是经常？

**第2轮 - 焦虑情绪**：
> 会不会经常担心这担心那，心里放不下？也是从不、很少、有时、还是经常？

**第3轮 - 孤独感**：
> 会不会感到孤独，觉得没人陪？同样选从不、很少、有时、还是经常。

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `depression` | 是否有抑郁感/情绪低落 | 文本 | 从不 / 很少 / 有时 / 经常 |
| `anxiety` | 是否有焦虑感 | 文本 | 从不 / 很少 / 有时 / 经常 |
| `loneliness` | 是否感到孤独 | 文本 | 从不 / 很少 / 有时 / 经常 |

**说明**：心理健康是老年人整体健康的重要组成部分。避免直接问"有没有抑郁症"这类专业词汇，改用生活化的表述，老人更容易真实回答。

---

### 第 8 部分：生活方式（G8）

**字段数**：4 个

**提问内容**（拆分为多轮）：

**第1轮 - 吸烟**：
> 吸烟吗？可以说从不、以前有现在没有、偶尔、还是经常？

**第2轮 - 饮酒**：
> 喝酒吗？同样可以说从不、以前有现在没有、偶尔、一周喝几次，或者其他情况。

**第3轮 - 锻炼**：
> 平时有锻炼吗？比如散步、做操这类。选从不、很少、有时、还是经常？

**第4轮 - 睡眠**：
> 睡眠质量怎么样？选很好、好、一般、差、还是很差？

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `smoking` | 吸烟情况 | 文本 | 从不 / 以前有现在没有 / 偶尔 / 经常 |
| `drinking` | 饮酒情况 | 文本 | 从不 / 以前有现在没有 / 偶尔 / 经常 |
| `exercise` | 锻炼情况 | 文本 | 从不 / 很少 / 有时 / 经常 |
| `sleep_quality` | 睡眠质量 | 文本 | 很好 / 好 / 一般 / 差 / 很差 |

**说明**：生活方式因素影响健康风险和疾病管理。支持用户用模糊词汇回答，如"以前有，现在没有"、"一周喝三五次"等，系统会自动理解和转换。

>

|--------|----------|----------|-------------|

| `height` | 身高（厘米） | 数字 | 如：160 |
| `vision` | 视力情况 | 文本 | 好 / 一般 / 差 |
| `hearing` | 听力情况 | 文本 | 好 / 一般 / 差 |

**说明**：身体指标用于计算 BMI 和评估感官功能。

### 第 10 部分：社会支持（G10）

**字段数**：6 个

**提问内容**：
> 最后了解一下家庭和支持情况：
> 
> ① 现在和谁住在一起？（独居/和老伴/和子女/和老伴及子女/住养老院）
> 
> ② 同住有几个人（不算自己）？
> 
> ③ 经济状况自己觉得怎么样？（很好/好/一般/差/很差）
> 
> ④ 有没有医保？是什么类型的？
> 
> ⑤ 平时主要是谁在照顾？（子女/老伴/保姆/自己/无人）

**需要收集的字段**：

| 字段名 | 中文名称 | 数据类型 | 可选值/说明 |
|--------|----------|----------|-------------|
| `living_arrangement` | 居住安排 | 文本 | 独居 / 与配偶同住 / 与子女同住 / 与配偶及子女同住 / 养老院 |
| `cohabitants` | 同住人数（不含本人） | 整数 | 如：2 |
| `financial_status` | 经济状况自评 | 文本 | 很好 / 好 / 一般 / 差 / 很差 |
| `income` | 年收入（元） | 数字 | 如：12000，不知道可留空 |
| `medical_insurance` | 医疗保险类型 | 文本 | 如：城镇职工医保 / 城乡居民医保 / 新农合 / 无 |
| `caregiver` | 主要照护者 | 文本 | 如：子女 / 配偶 / 保姆 / 无人照护 |

**说明**：社会支持系统对老年人的健康和照护质量有重要影响。

### 4.2 智能提取示例（支持模糊表述）

用户可以用自然语言、模糊词汇或一次性提供多个分组的信息：

## 五、数据使用说明

### 5.1 数据验证规则

- **必填字段**：所有 62 个字段都是必填的（系统会持续追问直到填完）
- **数据类型**：
  - 整数：age, education_years, cohabitants, weight, height, income
  - 文本：其他所有字段
  - 列表：cognition_calc（3个元素的列表）

### 5.2 数据用途

收集的数据用于：

1. **失能状态判定**：基于 BADL/IADL 评分
2. **健康风险预测**：短期和中期风险评估
3. **健康画像生成**：功能状态、优势、问题分析
4. **行动计划制定**：个性化照护建议
5. **优先级排序**：紧急程度分级
6. **报告生成**：完整的健康评估报告

## 六、优化建议（已实施）

### 6.1 已完成的优化

#### 1. 问题拆分与温和话术
- ✅ 第1部分：从一次性问6个问题，拆分为3轮提问
- ✅ 第6部分：从一次性问6个问题，拆分为6轮提问（每道题单独问）
- ✅ 第7部分：从一次性问3个问题，拆分为3轮提问
- ✅ 第8部分：从一次性问4个问题，拆分为4轮提问

#### 2. 用语优化（避免专业词汇）
- ✅ "受教育年限" → "读过几年书"
- ✅ "限制" → "影响您出门、活动或者做事"
- ✅ "不需要帮助" → "能自己来"
- ✅ "部分帮助" → "需要部分帮助"
- ✅ "多方面帮助" → "需要很多帮助"
- ✅ "情绪低落、提不起劲儿" → "心里闷闷的、高兴不起来"
- ✅ "紧张焦虑" → "经常担心这担心那，心里放不下"

#### 3. 支持模糊表述
- ✅ 吸烟/饮酒：支持"以前有现在没有"、"一周喝三五次"等模糊词汇
- ✅ BADL/IADL：支持用数字编号回答（如"1、2、3需要帮助"）
- ✅ 生活方式：支持"偶尔整点"、"基本都能做"等口语化表述

#### 4. 开场话术优化
- ✅ 避免提及"10个部分"，以免老年人觉得要填写很多内容而放弃
- ✅ 强调"按平时说话的方式回答"，降低心理负担
- ✅ 强调"记不清也没关系"，增加包容感

#### 5. 认知功能部分特殊优化
- ✅ 开场强调"很多老人都会有点记不清，这都很正常"
- ✅ 计算题改为"100减7"、"200减5"等具体数字，避免"再减一次"这类模糊表述
- ✅ 每道题单独提问，避免一次性问太多导致漏答

### 6.2 后续优化方向

1. **信息提取优化**：
   - 提升 LLM 的同义词识别能力
   - 增加对口语化表述的理解
   - 支持更多模糊匹配场景

2. **动态调整**：
   - 根据用户回答动态调整问题难度
   - 对认知功能有问题的用户，简化后续问题
   - 对健康状况良好的用户，可选填非关键信息

3. **进度反馈**：
   - 实时显示完成进度（但不强调"10个部分"）
   - 显示已完成的分组名称
   - 鼓励性反馈（"很好，继续"）

4. **错误恢复**：
   - 用户回答不清时，提供示例而非直接纠正
   - 支持用户修改之前的回答
   - 允许用户跳过不确定的问题（非关键字段）
